## 2. 整体架构（基于源码）

### 2.1 核心对象与职责
- `Simulation`：全局仿真容器，持有 `Topology` 和 `System`。
- `Topology`：拓扑图（节点、边、路由表）。路由默认通过 BFS 构建 next-hop。
- `System`：设备注册表，维护 `TopoID -> Device*`。
- `Device`：所有设备基类（Requester、Switch、DuplexBus、Snoop、DRAMsim3Interface）。
- `EventEngine`（在 `xerxes_standalone.cc` 中）：全局事件队列（按 Tick 的 multimap）。

### 2.2 运行主流程
入口 `main.cc` 的主流程：
1. 创建 `Simulation`，并调用 `init_sim()` 设置全局仿真对象。
2. 解析 TOML 配置（`parse_config()`）。
3. 为所有 Requester 注册初始 issue 事件。
4. 循环调用 `step()` 推进事件引擎；并按 `clock_granu` 推进 DRAMsim3 时钟。
5. 所有请求发完、队列排空后退出，输出统计。

### 2.3 配置解析与设备装配
`parse_config()` 会：
- 从 TOML 读取 `general.devices` 与 `general.edges`。
- 动态实例化设备（支持 `Requester/Switch/DuplexBus/DRAMsim3Interface/Snoop`）。
- 把设备加入 `System`，记录名字到 `TopoID` 映射。
- 根据 `edges` 建图，并调用 `topology->build_route()`。
- 把所有 DRAM 端点信息（起始地址、容量、写比例）注入每个 Requester 的 end points。

### 2.4 关键设备行为
- `Requester`：支持 `stream/random/hotcold/trace` 负载生成；内部有 issue queue 与 fake LRU cache；请求发出后经拓扑转发。
- `Switch`：多端口转发，端口内有按上游来源分队列与时间线调度，统计 switch queue delay/time。
- `DuplexBus`：支持全双工/半双工，建模方向切换开销（half duplex reverse）、帧封装和带宽延迟。
- `Snoop`：挂在内存前用于一致性相关行为，含可替换驱逐策略（LRU/FIFO/LFI 等）。
- `DRAMsim3Interface`：对接 DRAMsim3，负责请求接入、可受理检查、回调完成并发响应包。

### 2.5 数据与日志
- 包模型：`Packet`（请求类型、时戳、路径、统计字段）。
- 统计字段：bus/switch/snoop/DRAM 细分延迟，最终可汇总 `total_time`。
- 默认日志：CSV，每行一个请求，字段在 README 有完整解释。

## 3. 编译方法

### 3.1 官方 CMake 方式（README）
```bash
cd CXL_BETTER_SIM/Xerxes
mkdir -p build
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release -DCMAKE_EXPORT_COMPILE_COMMANDS=True
cmake --build build --target Xerxes
```

### 3.2 一键编译脚本（本次新增）
已新增 `build.sh`（位于 `CXL_BETTER_SIM/Xerxes/build.sh`）：
- 自动执行 `git submodule update --init --recursive`
- 自动创建 `output/` 和构建目录
- 默认 `Release` 构建
- 可通过环境变量调整

用法：
```bash
cd CXL_BETTER_SIM/Xerxes
./build.sh
```

可选参数与环境变量：
- 参数1：构建目录（默认 `build`）
- `BUILD_TYPE`：默认 `Release`
- `PARALLEL`：默认 `nproc`

示例：
```bash
BUILD_TYPE=Debug PARALLEL=16 ./build.sh debug
```

## 4. 清理编译产物方法

### 4.1 一键清理脚本（本次新增）
已新增 `clean.sh`（位于 `CXL_BETTER_SIM/Xerxes/clean.sh`），会清理：
- 顶层 CMake 构建产物（`build/`、`debug/`、`CMakeFiles/`、`CMakeCache.txt` 等）
- `DRAMsim3` 目录下常见 CMake 构建产物（`DRAMsim3/build` 等）

用法：
```bash
cd CXL_BETTER_SIM/Xerxes
./clean.sh
```

### 4.2 手工清理（备用）
```bash
rm -rf build debug CMakeFiles CMakeCache.txt cmake_install.cmake compile_commands.json Makefile Testing
rm -rf DRAMsim3/build DRAMsim3/CMakeFiles DRAMsim3/CMakeCache.txt DRAMsim3/cmake_install.cmake DRAMsim3/Makefile DRAMsim3/Testing
```

## 5. 实例用法

### 5.1 生成配置
```bash
cd CXL_BETTER_SIM/Xerxes
python3 configs/sample-bus.py --scale 4 --ratio 0.25 --log output/sample-bus.csv > configs/sample-bus.toml
python3 configs/sample-topo.py --scale 16 --topo ring --log output/sample-topo.csv > configs/sample-topo.toml
```

### 5.2 运行仿真
```bash
cd CXL_BETTER_SIM/Xerxes
./build.sh
build/Xerxes configs/sample-bus.toml
build/Xerxes configs/sample-topo.toml
```

### 5.3 结果分析
```bash
cd CXL_BETTER_SIM/Xerxes
python3 output/report.py output/sample-bus.csv --report=bw
python3 output/report.py output/sample-topo.csv --report=avg_lat
```

### 5.4 论文 AE 脚本
```bash
cd CXL_BETTER_SIM/Xerxes
bash AE-scripts/run_all.sh
```

## 6. 与 gem5 的关系
目录中包含 `gem5/` 适配代码（Ruby 协议、wrapper、示例配置），说明 Xerxes 既可 standalone 运行，也支持与 gem5 集成扩展；当前 `CMakeLists.txt` 的默认构建目标是 standalone 可执行 `Xerxes`。

## 7. 备注
- `init` 文件已提供 debug/release 双配置的 CMake 初始化命令。
- 若后续要做更细粒度性能分析，建议重点跟踪：
  - `Timeline::transfer_time()` 的排队模型
  - `DuplexBus` 半双工方向反转开销
  - `Snoop` 驱逐策略对失效流量影响
  - `DRAMsim3Interface` 的 issue/回调节拍对尾延迟影响
